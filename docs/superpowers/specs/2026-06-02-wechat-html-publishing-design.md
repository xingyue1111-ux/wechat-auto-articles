# 公众号 HTML 正文发布设计

## 背景

现有系统把文章排版为四张 1080px 宽 PNG 长图。用户将四张图片上传到微信公众号编辑器后，编辑器和预览阶段可以正常显示，但正式提交发布后正文图片消失。图片体积小于 2 MB，旧链路不再继续排查。

下一版的核心目标是把公众号正文从“图片承载全文”改为“原生 HTML 文字正文”，同时保留 Seedream 配图和现有长图导出能力。

## 目标

- 每天生成一篇适合公众号发布的完整文章。
- 正文约 1400-1600 个中文字符，围绕一个核心观点连续叙事。
- 生成 4 张 Seedream 配图，并嵌入公众号 HTML 正文。
- 后台提供“一键复制公众号正文”按钮。
- 复制后允许在公众号编辑器内继续微调。
- 外部图片无法稳定接管时，提供独立下载与手动替换退路。
- 保留现有四张 PNG 长图，作为备用导出。

## 非目标

- 第一版不调用公众号接口上传图片。
- 第一版不自动创建公众号草稿。
- 第一版不删除现有四张长图生成能力。
- 第一版不继续投入时间追查旧长图发布异常。
- 第一版不使用复杂 HTML、脚本、外部 CSS 或特殊字体。

## 文章内容结构

DeepSeek 生成一篇完整文章，而不是把 10 屏短文案作为主要交付内容。

文章包含：

1. 标题：一句明确判断。
2. 导语：解释今天为什么值得关注。
3. 第一部分：发生了什么。
4. 第二部分：为什么重要。
5. 第三部分：企业应该如何判断和行动。
6. 结尾：给出简短行动建议。

全文控制在 1400-1600 个中文字符附近。正文使用连续叙事，每个部分可以包含多个自然段。

## 配图位置

每篇文章固定生成 4 张 Seedream 配图：

1. 图 1：头图，放在导语后。
2. 图 2：放在“发生了什么”之后。
3. 图 3：放在“为什么重要”之后。
4. 图 4：放在行动建议之前。

图片只承担视觉节奏，不承载关键信息。图片缺失时，正文仍然完整可读。

## 后台交付

每天生成完成后，后台展示两组产物。

### 主要产物：公众号 HTML 正文

- HTML 文章预览。
- “一键复制公众号正文”按钮。
- 4 张配图的独立下载按钮。
- 每张图片显示编号和对应插入位置提示。

### 备用产物：四张 PNG 长图

- 保留现有 1080px 宽长图下载入口。
- 用于朋友圈分享、存档或其他特殊场景。
- 不再作为公众号正文的默认发布方式。

## HTML 兼容策略

复制内容只使用简单、内联的 HTML 样式：

- 普通段落。
- 小标题。
- 引用框。
- 分隔线。
- 图片。

不使用脚本、外部 CSS、复杂布局或特殊字体。公众号编辑器粘贴后，用户可以检查并做少量微调。

默认复制时带上 4 张外部图片。由于公众号是否稳定接管外部图片需要实际验证，后台必须同时提供 4 张图片下载入口。若某张图片异常，用户可以下载后在公众号编辑器中手动替换。

## 数据流

1. 系统抓取过去 24 小时内的公开信号。
2. DeepSeek 选择一条核心主线，生成 1400-1600 字完整文章。
3. 系统根据文章结构生成 4 条 Seedream 配图提示词。
4. Seedream 生成 4 张图片并持久化。
5. 系统生成公众号 HTML 正文，将图片插入固定位置。
6. 系统继续生成四张备用 PNG 长图。
7. 后台同时展示 HTML 预览、复制按钮、配图下载和长图下载。

## 错误处理

- DeepSeek 文章结构或长度不符合要求时，沿用现有重试策略。
- Seedream 生成或图片持久化连续失败时，中止本次生成，不发布不完整内容。
- HTML 预览必须在图片缺失时仍能显示完整正文。
- 后台复制失败时给出明确提示，不静默失败。

## 验收标准

第一版需要完成一次真实公众号发布测试：

1. 生成文章、4 张 Seedream 配图和 4 张备用长图。
2. 在后台点击“一键复制公众号正文”。
3. 粘贴到公众号编辑器。
4. 检查标题、段落、小标题、引用框、分隔线和 4 张图片。
5. 提交发布一篇测试文章。
6. 通过正式文章链接确认正文与 4 张图片仍然可见。
7. 若外部图片未被公众号稳定接管，验证下载图片并手动替换后可以正常发布。

## 后续升级方向

若第一版发布流程稳定，再评估公众号接口自动化：

- 上传图片到公众号素材系统。
- 自动创建公众号草稿。
- 减少手动检查和替换图片的步骤。
