### 生成流 300 秒超时修复

- 根因：DeepSeek 首次输出校验失败后重试，单次请求没有超时保护，可能几乎耗尽 Vercel 300 秒函数预算；流水线随后仍启动 Seedream，导致平台切断连接，前端只能显示“任务没有返回完成状态”。
- 修复：DeepSeek 请求增加 75 秒默认超时，并允许主流水线按剩余预算下调；主流水线增加总预算保护，Seedream 远程生成最多 105 秒，剩余不足 45 秒时跳过真请求并用占位配图收尾。
- 检查：新增 `tests/pipeline-time-budget.test.ts` 和 DeepSeek 超时信号测试；`npm.cmd run test:run` 通过，24 个测试文件、104 个测试；`npm.cmd run lint` 和 `npm.cmd run build --cache C:\tmp\npm-cache` 通过。
