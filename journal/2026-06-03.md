## 2026-06-03

### DeepSeek JSON 兜底问题
- 现象：线上生成时 DeepSeek 首次结构校验失败，第二次返回内容不是合法 JSON，系统降级为规则化兜底内容。
- 判断：新公众号正文要求 1400-1600 字，再加 10 个 panel、配图提示词和来源链接，原 `max_tokens: 5000` 容易导致 JSON 被截断。
- 处理：将 DeepSeek `max_tokens` 提高到 8000，并在 API 返回 `finish_reason: "length"` 时给出明确的“输出被截断”错误。
- 检查：`npm.cmd run test:run`、`npm.cmd run lint`、`npm.cmd run build` 均通过。

### 后续观察
- 线上再次生成时，如果仍降级，需要看日志是否显示“输出被截断”还是结构校验失败。
- 如果仍是结构校验失败，下一步应细分校验原因，明确提示是标题、panel 顺序还是字数不合格。

### 公众号发布后图片消失
- 现象：复制 HTML 到公众号后，编辑器预览可以显示图片，但正式发布后图片消失。
- 判断：公众号预览阶段能加载外部图片链接，不代表发布后会稳定保留外链图片。
- 处理：复制用 HTML 不再输出外链 `<img>`，改为输出“请在此处手动上传配图 X”的占位提示。
- 工作台文案改为提醒用户粘贴正文后手动上传 4 张配图，右侧继续提供 Seedream 图片下载入口。
- 检查：`npm.cmd run test:run`、`npm.cmd run lint`、`npm.cmd run build` 均通过。

### 连续几天标题相同排查
- 现象：后台列表中 2026-05-31 至 2026-06-03 多篇文章都显示“企业 AI 落地信号图”，用户怀疑新闻没有更新或主题素材太少。
- 判断：归档详情和生成日志显示候选源在更新，06-02、06-03 的来源和正文主题不同；主要问题是 DeepSeek 校验失败后进入规则兜底，而兜底顶层标题固定，后台列表也没有显示生成模式。
- 处理：兜底文章标题改为使用当天主素材的中文标题；后台已保存简报列表增加“DeepSeek 正文 / 规则兜底 / 生成模式未知”标识。
- 检查：`npm.cmd run test:run`、`npm.cmd run lint`、`npm.cmd run build --cache C:\tmp\npm-cache` 均通过。
